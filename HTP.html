Definition

Hypertext Transfer Protocol its a system that allows communication between web browsers and web servers.It is an application layer protocol designed to transfer infromation between networked devices and runs on top of other layers of the network stack.

When you visit a website, your browser uses HTTP to send a request to the server hosting that site, and the server sends back the data needed to display the page.

The working of HTTP

/
  • Open Web Browser: First, you open your web browser and type a website URL (e.g., www.example.com).
  • DNS Lookup: Your browser asks a Domain Name System (DNS) server to find out the IP address associated with that URL. Think of this as looking up the phone number of the website
  • Send HTTP Request: Once the browser has the website’s IP address, it sends an HTTP request to the server. The request asks the server for the resources needed to display the page
  • Server Response: The server processes your request and sends back an HTTP response. This response contains the requested resources
  • Rendering the Web Page: Your browser receives the data from the server and displays the webpage on your screen.

Role

  • Web Resource Transfer: HTTP facilitates the transfer of data such as text, images, and video from servers to clients, enabling the display of web pages.
  • Client-Server Communication: It acts as the intermediary, allowing browsers to send HTTP requests and servers to respond with data.
  • Webpage Publishing/Retrieval: It allows users to access web pages by initiating requests for specific URLs and receiving HTML content to render
