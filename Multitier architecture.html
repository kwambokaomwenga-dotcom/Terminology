Definition

Multitier architecture is a client-server software model that separates applications into distinct, physically separated layers—typically presentation, logic, and data—running on separate servers. This structure enhances scalability, security, and maintainability by allowing independent development, deployment, and scaling of each tier.

Roles from the 3 layers

Presentation Tier

This is the topmost level where users interact with the application. Its primary role is to collect user input and display processed data.

Application Tier

Often called the "heart" of the application, this tier processes the business logic, executes workflows, and makes logical decisions. It serves as a bridge between the presentation and data tiers.

Data Tier

Responsible for storing, managing, and retrieving the application's information.

Benefits/Features

  • Scalability - each tier can be scaled independently.
  • Maintainability - changes in one tier (like updating the user interface) often have minimal impact on the other tiers.
  • Security-by isolating the data tier from the front end, you reduce the risk of direct attacks on the database
  • Independent Development - different teams can work on different tiers simultaneously as long as the interfaces (APIs) between them are well-defined
